## 核心能力

### 双层凭证体系

平台采用**接口认证**和**游戏数据凭证**分离的设计：

**接口认证（三选一，证明调用权限）**

| 认证方式 | 请求头 | 适用场景 |
|---------|--------|---------|
| **Anonymous Token** | `X-Anonymous-Token` | 游客访问公开数据 |
| **JWT Access Token** | `Authorization: Bearer` | Web 平台登录用户 |
| **API Key** | `X-API-Key` | 第三方开发者/机器人 |

**游戏数据凭证（查询个人游戏数据时额外携带）**

| 凭证 | 请求头 | 说明 |
|------|--------|------|
| **Framework Token** | `X-Framework-Token` | 通过森空岛登录获取，关联游戏账号 |

## API 基础地址

```
https://api.end.shallow.ink
```

所有 API 端点均以此为基础路径。

## 响应格式

所有接口统一返回 JSON 格式：

```json 成功响应
{
  "code": 0,
  "message": "成功",
  "data": { ... }
}
```

```json 错误响应
{
  "code": 400,
  "message": "请求参数错误"
}
```
